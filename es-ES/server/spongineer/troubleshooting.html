Solución de problemas¶

Probablemente estés aquí porque algo salió mal con tu servidor de Sponge. Veamos si podemos descubrir qué, y qué hacer al respecto.

Potential Sources of Trouble

  • Java no está instalado en tu ordenador
  • Fallo en la conexión de red (o ataque DDoS)
  • No hay suficiente memoria libre
  • Malformed Config File (e.g. Bad Editing)
  • Un Complemento (o Modificación) Ha Funcionado Mal
  • Operating System Unstable (e.g. Virus Infection)
  • Datos corruptos
  • Problema entre el teclado y la silla
  • Hay una Falla en Sponge
  • Algo está Mal con el Universo

Java no está instalado en tu ordenador¶

Solution: Get Java. Visit the Instalación de Java for more information.

Fallo en la conexión de red (o ataque DDoS)¶

Sintomas: La conexión de red es muy lenta, va y viene, o es ausente.

Soluciones: Verifica tu conexión al módem o enrutador. Ve si tu navegador tiene problemas similares. Puedes usar un servicio gratuito como speedtest.net para verificar la velocidad de tu conexión. Otros servicios que se ejecutan en tu computadora o red local pueden ser la causa. Asegúrate de haber habilitado el reenvío de puertos en tu enrutador. Un ataque DDoS, aunque poco probable, probablemente mate por completo tu conectividad, y deberías ponerte en contacto con tu ISP si crees que este es el caso.

No hay suficiente memoria libre¶

Síntomas: El servidor se cierra inesperadamente, a menudo acompañado con mensajes «Out of Memory».

Solutions: Expand the maximum Perm memory size with the startup argument -XX:MaxPermSize=128. Expand your server heap memory (if possible) with startup arguments e.g. -Xms1024M (1GB starting memory) and -Xmx2048M (2GB maximum). Monitor your free memory on the computer and see if there is some locked up in other processes. You may need to kill frozen java processes, or restart your machine. Memory leaks sometimes occur with bugs in plugins, which can take time to isolate.

¿Sigue siendo un problema?: Si todavía tienes problemas a pesar de lo anterior y no puedes aumentar el Tamaño de Pila, verifica en tu Administrador de Tareas para ver si estás usando toda la Memoria disponible. Si es así, la única solución es agregar más RAM a tu sistema. Si todavía hay suficiente memoria disponible, estás ejecutando Java de 32 bits. Si estás utilizando Java de 32 bits, recomendamos una actualización a Java de 64 bits, siempre que su sistema operativo también sea de 64 bits.

Malformed Config File (e.g. Bad Editing)¶

Síntoma: Uno (o más) plugins no cargan o se comportan de un modo inesperado. Los archivos de log del servidor contendrás mensajes sobre ficheros que no se pueden leer al iniciar. El servidor podría cerrarse inesperadamente y los datos podrían estar corruptos.

Solución: Deten el servidor, y comprueba tus archivos editados. Carga copias de seguridad de todos los datos corrompidos. Podrías necesitar eliminar un archivo de configuración totalmente y permitirle regenerarse en el arranque del servidor.

Un Complemento (o Modificación) Ha Funcionado Mal¶

Síntoma: Podría ser casi cualquier cosa: lo que quiera que tu plugin haga más el factor x. Normalmente el servidor se cierra inesperadamente con una larga lista de mensajes de error en los archivos de log del servidor.

Solution: Stop the server, and check to see nothing has been corrupted. Be sure to check that it isn’t from an incorrectly edited config file (above). Remove suspect plugins and add them again one by one, restarting the server each time. The problem may originate from one plugin that is out of date - check for updates. Plugin conflict may also be the cause, having two incompatible plugins. You can also check our Mod Compatibility page to see if the problem is already known, and if there’s a way to fix it.

Operating System Unstable (e.g. Virus Infection)¶

Síntoma: el servidor sigue fallando o caducando, y otras partes de su sistema operativo también están teniendo problemas.

Solution: Stop everything. Thoroughly check your system and storage devices for malware and viruses. Good tools for this include AdwCleaner, Junkware Removal Tool, MalwareBytes, and most antivirus ware. Check your server files for corruption after a clean restart of your system. Examine the hardware for damage too if the problems persist - e.g. a faulty power supply.

Datos corruptos¶

Síntoma: Los archivos del mundo no se cargan o hacen que el servidor se cuelgue cuando los jugadores ingresan ciertos fragmentos. Corrupción de la base de datos.

Solución: Carga archivos de respaldo de datos dañados. El software para reparar mundos dañados está disponible y las regiones faltantes pueden regenerarse. Investigue la causa de la corrupción: ¿se trata de un complemento mal formado, un controlador de base de datos, una falla de energía u otra cosa? Siempre asegúrese de realizar copias de seguridad periódicas de datos importantes en un dispositivo seguro.

Problema entre el teclado y la silla¶

Síntoma: Todo funcoinaba perfectamente ayer. Hoy han empezado a pasar cosas extrañas desde que hice XYZ…

Solución: SpongeDocs no es lo suficientemente grande como para abarcar las cosas que las personas pueden hacer y que harán que el software falle de manera impredecible. Siempre vale la pena pensar detenidamente sobre lo que puede haber hecho recientemente que podría haber afectado el buen funcionamiento de su servidor. Una tarjeta de memoria puede estar suelta después de quitar el polvo, un acceso directo puede estar roto…

Hay una Falla en Sponge¶

Sintoma: Ninguno de los anteriores aplica, y aun no funciona como debería.

** Solución **: Es hora de sacar la artillería pesada. Presenta un informe en el seguimiento de problemas SpongeForge o SpongeVanilla, y recuerda incluir detalles de la versión de Forge y Sponge que estás utilizando, y un enlace al archivo de registro del servidor correspondiente.

Algo está Mal con el Universo¶

No podemos ayudarte con este problema. Te las tienes que apañar tú solo.
