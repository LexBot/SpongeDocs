论坛、Discord 及 IRC 守则¶

欢迎访问 Sponge 项目！很高兴你对 Sponge 感兴趣。我们有几个准则，适用于所有我们的交流工具，包括（但不限于）：

  • 官方 IRC 频道：#sponge，#spongedev，#spongedocs
  • Sponge 论坛
  • GitHub 及漏洞/问题跟踪系统
  • Sponge Discord
  • Sponge subreddit
  • Sponge TeamSpeak 服务器
  • Sponge 项目的其它官方交流工具。

Sponge 项目遵循着 Wheaton’s Law：“Don’t be a dick”。我们希望每个使用我们的交流工具的人都能理性，负责任，并尊重其它用户以及工作人员。

所有用户都应阅读并遵守我们的服务条款。

请继续阅读以下指引，但请明确这些只是一般的准则，而不是该做和不该做的详尽陈列。这些都应该是常识。

  • 有任何问题，请遵循支援指引（如果有）。
  • 请勿滥用 IRC、论坛和 Discord 等公众频道，换言之——请勿刷屏、水贴、反复在多个地方发表同样的内容等等等等。时刻记住：这些空间属于所有人，所以请务必礼貌待人。
  • 至少使你的言论通过 PG-13 级标准（译者注：PG-13 是美国电影分级标准中的一级，意为不建议十三岁以下儿童观看，特别是在没有家长陪同下观看）。我们并不是你的保姆，也并不会为你的脏话承担责任，所以请保持适当的礼节。
  • 禁止发布带有种族主义、性别歧视等下流肮脏的言论——这是条红线，我们也绝不允许任何程度的违反。
  • 请以友善的态度对待管理人员——他们都是 志愿者 ，时间有限。尽管他们大多数情况下都很愿意和你交流，但请勿大量发送私聊、在 Discord 和 IRC 上乱 @ 管理人员或进行其他骚扰。
  • 只有不能公开的东西才有必要私信管理人员。管理人员并不是你的私人搜索引擎。
  • 请尽量避免有关政治、宗教等有争议性的话题，因为他们不仅与 Sponge 没多大关系，而且还容易引发不必要的麻烦。
  • 不要无脑黑其他项目、 Mojang 的项目或与他们有联系的项目。
  • 不要总是问 Sponge 什么时候会好。它该好的时候会好，不该好的时候自然好不了。当它好了的时候，也不要急着求更新。更新在该出的时候自然会出。这也是一般的开源项目的基本准则之一。
  • 在 Sponge 的任何地方，以任何方式打广告的行为都是明令禁止的。这包括但不限于为你最喜欢的服务器运营商、你参与的服务器、你认为不错的一些软件、其他的游戏等任何东西打广告。有的时候会有一点点例外，不过就算是这些例外，在公开之前都会受到相关管理人员的审核。
  • 不要试图在 Sponge 的网站等系统进行任何金钱交易。这包括但不限于插件（以及 Mod 等类似物）、艺术作品、服务等。不过，我们特例允许在一个 发布帖 中的 Service Exchange 区域添加有关于招收开发者的信息，同时开发者也可以在 Devs Available 区域寻求相关的工作。其他任何有关金钱交易的行为都是明令禁止的。
    • 用户可以在“Service Exchange”区域发布（有工资或者没有工资的）招聘广告，不过所有发布相关内容的用户必须遵守发布模板及版规。
    • 用户可以在“Devs Available”区域发布应聘请求，不过所有发布相关内容的用户必须遵守发布模板及版规。
    • Service Exchange 版中仍然禁止付费插件。
  • 如果我们注意到 Sponge 平台托管的插件仅仅是一个收费插件的“轻量”免费试用版，我们有权利将其从插件托管列表中移除。至于一个插件到底是不是上述的“轻量”免费试用版，我们的管理员将予以定夺。
  • 我们保留使用 Sponge 用于描述官方（SpongePowered）项目的权利。请不要使用 Sponge 作为你的插件名称的一部分，除非：（1）你的插件也有依赖其他 API 的版本（在这种情况下， 可以在标题中加入诸如“For Sponge”等字样），（2）你的插件主要涉及到 Minecraft 方块“海绵（Sponge）”。
  • 我们的目标是完全符合 Mojang 的最终用户许可协议（EULA），因此所有被怀疑违反 EULA 的插件、服务、文章、链接等都会在 Sponge 团队人员的考虑后或者在 Mojang AB 的要求下被移除。
  • 不要对 Sponge 项目、其团队成员、附属人员组织或合作者等实施造谣、诽谤等会对 Sponge 项目带来不良影响的行为。
  • 我们的所有封禁行为都并非空穴来风。所以请不要逃避或试图绕过封禁行为。
  • 各种社交平台上的管理员等，拥有所有和 Sponge 项目相关的争端的最终解释和定夺权。

注解

上述的任何规定会在没有任何警告的情况下发生变更。任何被发现违反规定的内容，即使该内容创建于规定创建之前，也可以经过批准移除。这些规定普遍适用于 Sponge 对外交流的任何方式，但每一个独立的子方式（例如 subreddit 或者论坛版块），可能会有其自身的一套凌驾于这些一般规定之上的准则或附加规则。如果这样，每一套规则都应遵守。规则和准则会随着 Sponge 社区的发展而发展。它们仅作为指示使用；过多的使用 FLARD 有可能毁掉你的袜子。
