Débugger Sponge¶

Astuce

Consultez Debugging d’un plugin pour débugger un plugin et Utiliser le Debugger pour quelques astuces.

Ces sections vont vous indiquer comment débugger Sponge en lui-même. Peut importe si vous voulez débugger SpongeForge, SpongeVanilla, SpongeCommon ou SpongeAPI, vous devrez mettre en place un espace de travail SpongeForge ou SpongeVanilla. Vous débuggerez SpongeCommon et SpongeAPI avec l’implémentation.

Voir La structure du projet Sponge pour plus d’informations.

Note

Assurez-vous d’avoir lu et comprit Comment utiliser Git(Hub) et Organisation des branches du dépôt ainsi que la totalité des sections de Développer Sponge avant de continuer.

Mettre en place l’espace de travail¶

Voilà les étapes nécessaire pour mettre en place votre espace de travail si vous n’en avez pas déjà un:

  1. Clonez l’implémentation.
  2. Faites des changements dans votre dépôt local si besoin.
  3. Configurez l’implémentation.
  4. Compilez l’implémentation.
  5. Exécutez l’implémentation.

Une fois tout ça de fait, vous pouvez importer le projet dans votre IDE.

Note

Ce n’est qu’un aperçu des étapes nécessaire pour configurer l’espace de travail. Lisez le README de SpongeForge ou SpongeVanilla pour des instructions détaillées.
